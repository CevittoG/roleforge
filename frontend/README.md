# Frontend (Next.js static export, pages router)

Two views, both backed by the audit Sheet via the backend API:

- **Generate** (`/`) — paste JD text or a URL → `POST /api/generate`.
  On `409` (duplicate Company+Role), show a confirm dialog and re-POST with
  `confirm_overwrite: true`.
- **History** (`/history/`) — `GET /api/applications` → list sortable by date
  or `fit_score`. Each card links to the Drive folder (`folder_url`) and
  offers in-app downloads via
  `GET /api/download?folder_id=<id>&file=<resume|cover_letter|job_description|interview_prep>`.

Stack: Next.js 14 (pages router), TypeScript, Tailwind, Radix primitives
(Dialog, Tabs), lucide-react. UI components are hand-authored under
`src/components/ui/` — no shadcn CLI dependency.

## Why pages router?

The backend ships a strict CSP: `script-src 'self'`. Pages router emits only
external script chunks (plus `__NEXT_DATA__` which is `type="application/json"`
and not executable). App router emits inline `<script>` tags during hydration
that would require nonce/hash injection or relaxing the CSP. Stay on pages
router unless that changes.

## Dev workflow

Two terminals:

```bash
# 1. Backend (port 8000)
make run

# 2. Frontend (port 3000, proxies /api/* → 8000)
cd frontend
npm install        # first time only
npm run dev -- --hostname 0.0.0.0
```

`--hostname 0.0.0.0` makes the dev server reachable from your phone on the
LAN (`http://<mac-ip>:3000`) — essential because mobile is the primary device.

## Build (static export)

```bash
cd frontend
npm run build
```

`next.config.js` sets `output: 'export'` and `trailingSlash: true`, so the
build emits `./out/`. The Dockerfile copies that into `app/static/`; in
production FastAPI serves the static assets at `/` and the API at `/api/*`
from a single process (no CORS).

For local prod-path verification:

```bash
rm -rf ../app/static && mkdir -p ../app/static
cp -r out/. ../app/static/
cd .. && make run     # serves the built bundle through FastAPI
```

`app/static/` is gitignored — the build output is regenerated by the
Dockerfile in CI.

## Type sharing

`src/lib/types.ts` is hand-mirrored from `app/web/schemas.py`. The surface is
small enough that codegen isn't worth the tooling. If you change a schema
field on the backend, mirror it here too.

## Security

- Same-origin only (`fetch('/api/...')`). No env-baked URL, no secrets in the
  client. Cloudflare Access gates traffic at the edge in production.
- Never `dangerouslySetInnerHTML`. All model output, scraped JD text, and
  audit-log fields go into JSX text nodes (React escapes them).

## Mobile-first

Designed for ~360–412px viewports. Touch targets are ≥44px (`min-h-touch` and
`min-w-touch` utilities). The bottom nav respects `env(safe-area-inset-bottom)`
for notched devices.
